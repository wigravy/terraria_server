# Terraria tModLoader Server for Ubuntu

Язык:

- English: [../README.md](../README.md)
- Russian: [README.ru.md](./README.ru.md)

Простой мультиплеерный сервер `Terraria` на Ubuntu с использованием `tModLoader`.

Этот репозиторий устанавливает и запускает выделенный сервер `tModLoader` через SteamCMD только с анонимным входом.

## Что есть в репозитории

- shell-скрипты для установки и запуска
- `.env.example` со всеми поддерживаемыми настройками сервера
- документация по примерной сборке модов

В репозитории нет файлов Terraria, `tModLoader`, модов или исходного кода игры.

## Быстрый старт

1. Скопируйте репозиторий на Ubuntu-сервер.
2. Создайте локальный конфиг:

```bash
cp .env.example .env
```

3. При необходимости измените `.env`.
4. Сделайте скрипты исполняемыми:

```bash
chmod +x setup.sh start-server.sh
```

5. Запустите установку:

```bash
./setup.sh
```

6. Положите ваши `.tmod` файлы в `server_data/Mods`.
7. Если у вас есть экспортированный из локального tModLoader `install.txt`, положите его туда же.
8. Если `server_data/Mods/enabled.json` отсутствует, запустите `./setup.sh` еще раз, чтобы создать его по локальным `.tmod` файлам.
9. Запустите сервер:

```bash
./start-server.sh
```

## Структура данных

По умолчанию все рабочие файлы сервера хранятся в `./server_data` рядом со скриптами. При необходимости можно изменить только корневой путь `SERVER_DATA` в `.env`.

Скрипты создают и используют внутри `SERVER_DATA` такие папки:

- `Worlds`
- `Mods`
- `tmodloader`
- `steamcmd`

Файлы `serverconfig.txt`, `cli-argsConfig.txt` и `banlist.txt` также создаются внутри `SERVER_DATA`.

## Моды

Скрипты не скачивают моды.

Ожидаемый сценарий:

- поместите `.tmod` файлы в `SERVER_DATA/Mods`
- при необходимости добавьте туда же ваш `install.txt`
- если у вас уже есть `enabled.json`, скрипты его не перезаписывают
- если `enabled.json` отсутствует, `setup.sh` создает его по именам `.tmod` файлов

## Обновление

Повторный запуск `./setup.sh` обновляет установленные файлы `tModLoader` в `SERVER_DATA/tmodloader`.

По умолчанию SteamCMD устанавливает последнюю публичную ветку. Если указать `TMODLOADER_BRANCH`, будет запрошена эта ветка.

## Конфигурация

`.env.example` содержит:

- `SERVER_DATA` для корневой директории данных
- параметры установки, такие как `TML_APP_ID`, `TMODLOADER_BRANCH` и `MODPACK`
- все официальные переменные конфигурации выделенного сервера Terraria со значениями по умолчанию и комментариями

`worldpath` и `modpath` намеренно не выносятся в пользовательские переменные. Скрипты всегда держат их внутри `SERVER_DATA`.
